Git

git - version control system, manage source code history (local).

repository - the location where your code history is stored.

branch - “folder” within the repository containing commits.

Git helps control and manage folders/files.

Git can be accessed through the terminal.

Commands

git init [repository name]

This command is used to start a new repository.

git add .

It tells Git that you want to include updates to a particular file in the next commit.

git status

This command lists all the files that have to be committed.

git commit -m “name for the commit”

This command records or snapshots the file permanently in the version history.

git log

This command displays all of the commits in a repository's history.

git reset --hard[id of the git you want to go back to]git reset --hard[id of the git you want to go back to]

This command clears the mess from the index file and the working tree.

git checkout -- .

The git checkout command is used to update the state of the repository to a specific point in the project's history.

git branch

This command helps you create, delete, and list branches.

git checkout -b [name of branch]

This command creates and accesses the new branch.

git branch -D [name of the branch]

This command deletes the branch.

git merge

This command is used to integrate changes from another branch.

Github

github - git repository hosting (online)

Commands

git remote add origin [link]

Establishes a connection between a local and an online repository.

git remote

This command is one piece of the broader system which is responsible for syncing changes. Records registered through the git remote command are used in conjunction with the git fetch , git push , and git pull commands.

git remote -v

Shows where the repository is being pushed to (shows link).

git push

This command is used to upload local repository content to a remote repository.

git clone [repository link]

Clones all the contents to local repository

git fetch

This is a primary command used to download contents from a remote repository.

git pull

Fetches code from remote branch and merges with a local branch.


You can delete a repository on github from the repository’s settings by going to the danger zone on the bottom of the settings page.
